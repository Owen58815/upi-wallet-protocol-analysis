# 印度/东南亚UPI钱包协议研究

## UPI钱包的协议通常包含：

0. **初始化协议**：提供 APP 初始化、各项风控检测、初始秘钥获取等工作。  
1. **OTP 登录协议**：根据提供的 OTP 进行登录，返回 Token，维护 Token 状态。  
2. **查询 UPI ID 协议**：根据登录返回的 Token 查询 UPI ID。  
3. **查询流水协议**：根据登录返回的 Token 查询交易流水。  
4. **查询流水详情协议**：根据登录返回的 Token 查询流水详情。  
5. **登录状态**：通常根据 Token 是否存在以及有效期来判断。  

---

## 获取协议涉及到的技术：

1. **风控对抗**：在分析一些算法前，应对各种风控监测，各层级反编译、反抓包的对抗。  
   - 设备环境检测：是否 Root、是否开启调试模式、是否为定制 ROM、是否集成 Google Play  
   - 模拟器、VPN、WiFi 代理检测  
   - 谷歌设备完整性校验  
   - 对抗反抓包策略  
   - 服务器 SSL 证书校验等  

2. **协议获取**：在众多交互中获取关键的 0-5 协议接口、参数、参数个数、参数对应的加密算法，以及协议的返回结构等。  

3. **加密解密算法**：各种标准或非常规加密算法，如  
   AES / DES / RSA / HMC / Base64 / MD5 / Sha-1 / 256，以及自定义数据加密格式。  
   有的需要深入到 SO 分析加密逻辑，代码脱壳、去除混淆，通过分析还原代码，推导出参数算法。  

4. **文档与 Demo 输出**：基于分析到的协议与参数进行文档编写，汇总并输出与文档一致的 Demo。  

---

## 协议接入方案：

1. 可以基于给到的协议，在服务器配置代理池，实现 Java / Python 版本的登录、查询服务。  
   为了更好应对风控，需要构建一套自己的 IP 切换与设备参数模拟机制。  

2. 基于协议分析结果，唤醒支付，开发定制版钱包 App（在原钱包 App 嵌入服务插件），  
   实现将业务流水数据提交到自己的服务器。
