[UPI钱包协议研究]

通常印度/东南亚UPI钱包协议包含：

0，初始化协议： 提供APP初始化，各项风控检测，初始秘钥获取等工作。
1，OTP登录协议： 根据提供的OTP进行登录，返回Token，维护Token状态。
2，查询UPI ID协议： 根据登录返回的token查询upi id.
3，查询流水协议: 根据登录返回的token查询交易流水
4，查询流水详情协议：根据登录返回的token查询流水详情.
5，登录状态：通常根据token是否存在以及有效期来判断。

获取协议涉及到的技术：

1，风控对抗，在分析一些，算法钱，应对各种风控监测，各层级反编译，反抓包的对抗。
设备环境检测，是否root，开启调试模式，定制ROM， 是否集成google play；模拟器，VPN，wifi代理检测
。谷歌设备完整性校验，对抗反抓包策略，服务器SSL证书校验等。
2，协议获取，在众多交互中获取关键的0-5协议接口，参数，参数个数，参数对应的加密算法。协议的返回协议等
3，加密解密算法：各种标准的/非常规加密算法，如AES/DES/RSA/HMC/Base64/MD5/Sha-1/256,以及自定义的
数据加密格式。有的需要深入到SO分析加密，代码脱壳，去掉混淆，通过分析的还原代码，推导出参数算法。
4，基于分析到的协议，参数进行文档编写，汇总并输出和文档一致的Demo。

协议接入方案：

1，可以基于给到的协议，在服务器配置代理池，实现Java/Python版本的登录、查询服务。为了更好应对风控需要构建一套自己的ip切换，设备参数模拟的机制。
2，基于协议分析结果，可以开发定制版的钱包app（在原钱包app嵌入服务插件），实现将业务流水数据提交到自己的服务器。
